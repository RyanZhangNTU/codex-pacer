# Security Policy

## Supported versions

Security fixes are currently provided for the following release line:

- `1.0.x`

## Reporting a vulnerability

Please do not open a public GitHub issue with exploit details.

Preferred disclosure path:

1. Use GitHub's private vulnerability reporting workflow for this repository if it is enabled.
2. Use another private maintainer contact path if one is available for this repository.
3. If no private reporting or contact path is available, open a minimal public issue requesting a private contact channel and do not include sensitive details.
4. Include affected version, platform, reproduction details, and potential impact in the private report.

## What to expect

Maintainers will aim to:

- acknowledge valid reports promptly
- assess severity and affected scope
- prepare a patch or mitigation in an appropriate release
- credit reporters when appropriate and requested
